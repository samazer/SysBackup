00217 /**
00218  *
00219  * Do we want to make use of hardlinks between different backupsets?
00220  *
00221  * The current backup (being generated) should be on the same
00222  * partition as the backup that was generated by the most recent
00223  * successful backup run. (This software makes this assumption.) If
00224  * that's the case - ie: if the most recent successful run can still
00225  * be found on the same partition - it makes sense to use hardlinks
00226  * for identical files so as to save disk space. 
00227  *
00228  * Obviously this won't work if Keep Local is off or if there's a
00229  * separate script that moves the most recent backup off to another
00230  * partition.
00231  *
00232  * NOTES: 
00233  *
00234  * 1. The software will not handle attempts to hardlink identical
00235  *    files that are on different partitions.
00236  * 
00237  * 2. It is possible, over a period of many months or a few years, for
00238  *    normal hard disk bit rot to cause the data within very old files
00239  *    to deteriorate. In some cases (such as long term storage of
00240  *    photos, for example,) this will eventually become an
00241  *    issue. Please make sure that files are scanned every few months
00242  *    to ensure that the hard disk controller detects and correct any
00243  *    possible problems.
00244  *
00245  */
00246 $this->set_usehardlinks( true );
