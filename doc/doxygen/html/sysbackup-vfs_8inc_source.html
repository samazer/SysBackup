00002 /** 
00003  *
00004  * Virtual Backup Filesystem plugin
00005  *
00006  * <p>The backup_vfs_base_c class defines a basic set of file handling
00007  * methods for use by the various plugins. The idea is that all
00008  * plugins will use the same set of simple methods to move files and
00009  * tarballs of files into the backup set. Those methods will then deal
00010  * with the issue of copying the files into the local, remote or both
00011  * file stores. The plugins therefore don't need to know anything
00012  * about the backup set - they only need to know about the job they
00013  * are trying to perform.
00014  *
00015  * <p><b>Procedure</b>
00016  *
00017  * <p>The basic procedure will be as described below. The backup
00018  * system will instantiate a master class that will keep track of all
00019  * target filesystems to be used see the notes below about VM's that
00020  * backup to dom0 as well as a separate NAS device.) Each different
00021  * filesystem will be managed by a class that will be able to do the
00022  * following:
00023  *
00024  * <ul> 
00025  *
00026  * <li>The constructor will:
00027  *
00028  * <p>- Determine the location of the temporary workspace<br>
00029  * <br>
00030  * -Determine the name and location of the backup (the target
00031  *  folder for the backup being produced by the current program
00032  *    run.)<br>
00033  *<br>
00034  * - Find the name and location of the previous successful
00035  *    backup, if any.<br>
00036  *<br>
00037  * - Get the hashlist for the files that were written to the last
00038  *    successful backup (allowing the use of hardlinks to existing
00039  *    files in cases where those files are identical.)<br>
00040  *<br>
00041  *
00042  * <li>The class will provide a method for adding a sub-folder within
00043  * the backup set and selecting that sub-folder for use during a given
00044  * backup job.</li>
00045  *
00046  * <li>The class will provide a method for compressing output piped
00047  * from a process and adding it to the backup set.</li>
00048  *
00049  * <li>The class will provide a method for compressing a file on the
00050  * local filesystem and adding it to the backup set.</li>
00051  *
00052  * <li>The class will provide a method for tarballing a folder on the
00053  * local filesystem and adding it to the backup set.</li>
00054  *
00055  * <li>The class will list separately the tarballs that are hardlinked
00056  * so that future code can check for bit-rot.</li>
00057  *
00058  * <li>The class will keep track of available disk space on the target
00059  * filesystem and on the local filesystem. The class will also keep
00060  * track of all items added to the backup set. If disk space on the
00061  * local filesystem is limited the class, when writing to a remote
00062  * filesystem, the class will copy the file immediately to allow the
00063  * local copy to be deleted. If a full backup is being saved to the
00064  * local filesystem the class will queue files for remote copy and
00065  * perform the copies after all other backup tasks have been
00066  * completed.</li>
00067  *
00068  * <li>The class will implement a set of counters that will keep track
00069  * of disk space used, number of files written, number of files
00070  * hardlinked and any related information.</li>
00071  *
00072  * <li>The class will provide a number of methods for use by the
00073  * backup rotation plugin (create folders, list folders and recursive
00074  * delete of folders.)</li>
00075  *
00076  * </ul>
00077  *
00078  * <p>Note that this backup software will often be running on a Virtual
00079  * Machine with limited available disk storage. In many cases the VM
00080  * will need to backup directly to the dom0 while using the minimum
00081  * possible disk space on the VM. However, the dom0 is not suitable
00082  * for use as remote backup storage. In most cases the backup script
00083  * will also be forced to send the same backup set to a NAS device on
00084  * the network. Finally, after creating the full backup set on the
00085  * dom0 and the NAS device, this script will also need to trigger a
00086  * synchronization process between the local NAS device and an offsite
00087  * storage server. 
00088  *
00089  * <p>As such the backup VFS must be able to operate on the local
00090  * filesystem as well as two different remote filesystems.

00314    private function _remote_xferfile( $target, $hash, $size )
00315    {
00316       foreach( $this->sys_remotes as $id )
00317       {
00318          $start = microtime(true);
00319          if ( $this->notquiet() )
00320             $this->tee( "  {$id}:", "" );
00321 
00322          $r   = $this->sys_remote[$id]->remote_copyfile( $target, $hash, $size );
00323 
00324          if ( $this->sys_remote_weekly[$id]->plugin_enabled )
00325             $ref = $this->sys_remote_weekly[$id]->rotation_copyfile( $target, $hash, $size );
00326 
00327          if ( $this->sys_remote_monthly[$id]->plugin_enabled )
00328             $ref = $this->sys_remote_monthly[$id]->rotation_copyfile( $target, $hash, $size );
00329 
00330          $tim = microtime(true) - $start;
00331          $dur = $this->duration($tim);
00332          $hl  = $this->sys_remote[$id]->is_hardlinked( $target );
00333 
00334          if ( $this->notquiet() )
00335             $this->tee( ($hl ? "*" : "") . "{$dur}s" . ($r ? "??" : ""), "" );
00336       }
00337    }

00362       {
00363          $this->_notquiet_report_header();

00365          $fn   = str_pad( $finfo[0], $fname_width );
00366          $size = str_pad( si_suffix($finfo[2], "B"), 8, " ", STR_PAD_LEFT );
00367          $this->tee( "    {$size} {$fn}", "" );
00368 
00369          $this->_remote_xferfile( $finfo[0], $finfo[1], $finfo[2] );
