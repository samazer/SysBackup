00003  *
00004  * General tarball support
00005  *
00006  * This class is used to generate tarballs. The class is designed to
00007  * allow the calling software to specify a tool for making tarballs
00008  * (ie: usually tar or pax,) and, later, change the choice without
00009  * having to make any other changes to the calling software.
00010  *
00011  * Note the use of a separate gzip filter rather than using the
00012  * compression built into the tar/pax utility. This is because gzip
00013  * and other filters normally embed a time stamp and file name into
00014  * their output streams. As a result of this timestamp it is
00015  * impossible for two tarballs of the same collection of files to have
00016  * the same sha1 hash. To avoid this problem the tarball is generated
00017  * as uncompressed data and piped through the filter using an option
00018  * to suppress the timestamp info.
00019  *

00021  * utility, v2.

00468    private function _eval_funny_exitcodes( $r )
00469    {
00470       if ( $r["exitcode"] )
00471          $this->last_error( trim($r["stderr"]), true );
00472 
00473       /**
00474        * gzip is supposed to return a standard error result (ie: 0 =
00475        * good, non-zero = bad.) 
00476        *
00477        * Unfortunately the practical experience is that exit codes are
00478        * not as reliable as they should be. At the time of this
00479        * writing the reason for the problems is not known. Sometimes
00480        * the exitcode is -1 even though the filter completed
00481        * successfully.
00482        *
00483        * Another observation: Usually the proc_close return value is
00484        * -1 and the process exit code is 0. In many cases these values
00485        * seem to be reversed for no apparent reason.
00486        */
00487 
00488       if ( ($r["proc_close"] == -1)
00489         && ($r["exitcode"] == 0) )
00490          return 0;
00491 
00492       if ( ($r["proc_close"] == 0)
00493         && ($r["exitcode"] == -1) )
00494          return 0;
00495 
00496       if ( ($r["stderr"] == "")
00497         && ($r["exitcode"] == -1) 
00498          )
00499          return 0;
00500 
00501       return $r["exitcode"];
00502    }
