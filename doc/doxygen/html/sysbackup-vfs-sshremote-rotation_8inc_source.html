00005  *
00006  * Note that these rotation plugins are not actually loaded as plugins

00009  *
00010  * See the notes in the header of the localio rotation plugin.
00011  * This plugin is intended to be identical but operates
00012  * on a remote filesystem.
00013  *
00014  * This plugin is called after a file has been copied to the ssh
00015  * remote NAS server. It is intended to determine how best to create a
00016  * weekly or monthly rotation. Each rotation is actually a separate
00017  * backupset in the case where hardlinks are not available - so the
00018  * code operates as if it is creating a new backupset. 
00019  *
00020  * The situation is quite simple if it is possible to use hardlinks
00021  * between the daily rotations and the weekly rotations. In this case
00022  * the necessary folders are created and hardlinks are written as the
00023  * backup progresses.
00024  *
00025  * Otherwise the rotation plugin loads the hashlist for the previously
00026  * generated backupset in the rotation. It attempts to save disk space
00027  * via hardlinks between files in the new weekly backupset and the
00028  * previous weekly backupset using the same logic as the local I/O
00029  * backup plugin. If no identical file is found the remote server is
00030  * instructed to make a copy of the file from the daily rotation to
00031  * the weekly or monthly backupset.
